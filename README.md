# Metro Relic

Оффлайн пошаговый рогалик с полем 7×7: этажи, враги, выход, реликвии. Ядро — веб (`web/`), обёртка для iPhone — Capacitor (`ios/`). Статическая веб-сборка для GitHub Pages — каталог `docs/`.

## Возможности

- Автосохранение прогресса в браузере (`localStorage`).
- Настройки: масштаб UI, сложность, звук и громкость.
- Экраны: как играть, частые вопросы, что нового, статистика, форма «Сообщить о баге».
- Звуковые эффекты из файлов в `web/assets/sfx/`.
- Управление в вебе: свайп по полю и клавиши стрелок / WASD (тапы по клеткам для хода отключены).
- В iOS-приложении — свайп и остальная логика того же `web/`.

## Требования

- Node.js **22** или новее (требование Capacitor CLI 8.x; для локального `npm run dev` и `npx cap`).
- macOS + Xcode — для сборки и установки на iPhone.

## Быстрый старт (веб локально)

```bash
npm install
npm run dev
```

Юнит-тесты чистых функций (Node 22+):

```bash
npm test
```

На GitHub при push/PR в `main` или `master` (и вручную через **Actions → CI → Run workflow**) запускается **CI** (`.github/workflows/ci.yml`): `npm ci`, `npm test`, проверка синтаксиса JS и базовая валидация `web/index.html`. Локально без установки зависимостей из lock-файла: `npm run verify:ci`.

Открой в браузере: `http://localhost:4173`

Индикатор `Audio: …` в шапке показывается только на `localhost` / `127.0.0.1` или если открыть игру с параметром **`?debug=1`** (или `?audioDebug=1`). В сборке на GitHub Pages и в нативном iOS (Capacitor) он скрыт.

## Сборка и запуск на iOS

```bash
npm install
npx cap sync ios
npx cap open ios
```

В Xcode: выбери свою команду (Team), при необходимости смени Bundle Identifier, подключи iPhone и нажми Run. **На iPhone нативное приложение закреплено за портретной ориентацией** (`Info.plist`); гибридная вёрстка для ландшафта остаётся актуальной в **браузере** и на **iPad**.

Подробнее по шагам для новичков см. файл `README_IOS_SETUP_RU.md`.

## Публичная веб-версия (бета по ссылке)

### Автодеплой с GitHub Actions (рекомендуется)

В репозитории есть workflow **Deploy Pages** (`.github/workflows/deploy-pages.yml`): при каждом push в `main` или `master` выполняются `npm ci`, `npm test`, затем `sync-web-to-docs` и выкладка каталога **`docs/`** на GitHub Pages.

**Однократная настройка в GitHub:**

1. **Settings** → **Pages** → **Build and deployment**.
2. В поле **Source** выбери **GitHub Actions** (не «Deploy from a branch»). Иначе workflow не сможет опубликовать сайт.
3. После первого успешного прогона workflow ссылка будет вида  
   `https://<твой-логин>.github.io/<имя-репозитория>/`  
   (для организации — `https://<организация>.github.io/<репозиторий>/`). Эту ссылку можно отправлять внешним бета-тестерам: грузятся только статические файлы и скрипты, прогресс хранится в `localStorage` браузера. Полноценная оффлайн-игра без сети в Safari гарантируется в **нативной** сборке (TestFlight/App Store), где веб-сборка упакована в приложение.

При необходимости деплой можно запустить вручную: **Actions** → **Deploy Pages** → **Run workflow**.

### Локальная синхронизация `web/` → `docs/`

То же делает скрипт вручную (удобно перед коммитом, если хочешь держать `docs/` в git зеркально с `web/`):

```bash
npm run sync:docs
```

Скрипт копирует файлы и подставляет в `index.html` параметр `?v=` из поля **`assetVersion`** в `package.json` (и в `web/`, и в `docs/`). При изменении статики подними `assetVersion`, затем снова `npm run sync:docs` (и push — если деплой идёт через Actions, при следующем push артефакт соберётся заново).

В каталоге `web/` есть файл **`.nojekyll`**, он попадает в `docs/` и отключает обработку Jekyll на GitHub Pages (чтобы не ломались пути к ассетам).

### Нативное приложение на iPhone для тестеров

Веб по ссылке и приложение из App Store / TestFlight — разные каналы. Чтобы тестеры **установили приложение на iPhone**, нужен **TestFlight** (или распространение сборки вручную через Xcode на зарегистрированные устройства). Пошагово см. **`README_IOS_SETUP_RU.md`** (раздел про TestFlight и внешних тестеров).

## Структура репозитория

| Каталог / файл | Назначение |
|----------------|------------|
| `web/` | Игра: `index.html`, `game-config.js`, `locale-bundles.js`, `pure-grid.js`, `styles.css`, `game.js`, ассеты |

Язык интерфейса: **Настройки → Язык** (сохраняется в `localStorage`). Доступны **русский** и **английский**; тексты инструкций, логов и кнопок берутся из `locale-bundles.js`.
| `docs/` | Копия для GitHub Pages |
| `ios/` | Проект Xcode (Capacitor) |
| `capacitor.config.ts` | Конфигурация Capacitor |
| `web/victory-sfx-presets.js` | Пресеты победного звука |

## Звуки и медиа

Файлы в `web/assets/sfx/` должны распространяться и использоваться в соответствии с теми правами, которые у вас есть на эти файлы (собственная запись, покупка лицензии, разрешение автора и т.д.). Код репозитория лицензирован отдельно; сами аудиофайлы в репозиторий вы добавляете на свой страх и риск.
