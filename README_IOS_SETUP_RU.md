# Metro Relic — iOS и бета-тестирование

Готовый стартовый проект оффлайн-игры с веб-ядром и обёрткой iOS через Capacitor.

## Веб по ссылке или приложение на iPhone

| Канал | Что нужно | Как получить |
|--------|-----------|----------------|
| **Браузер** (внешние тестеры) | Репозиторий на GitHub, включённый **GitHub Pages** через **GitHub Actions** | Ссылка вида `https://<логин>.github.io/<репозиторий>/` — см. основной `README.md`, раздел «Публичная веб-версия» |
| **Приложение** на iPhone | Платная подписка **Apple Developer Program**, **App Store Connect**, сборка в **Xcode** | **TestFlight** (удобнее всего для внешних тестеров) — инструкция ниже |

Игра в `web/` одна и та же; Capacitor упаковывает её в нативный бинарник.

## Что уже сделано в проекте

- Играбельный оффлайн-прототип (`web/`): сетка, враги, этажи, реликвии.
- Конфиг Capacitor (`capacitor.config.ts`) — поле **`appId`** должно совпадать с **Bundle Identifier** в Xcode (сейчас: `com.tsvetpet.metorelic.game`).
- Проект Xcode в каталоге `ios/`.

## Быстрый старт (разработка)

1. Установить Node.js **22+** (нужно для Capacitor CLI 8.x).
2. В **корне** репозитория (рядом с `package.json`), не внутри `web/`:

```bash
npm install
npm run sync:ios
npm run open:ios
```

Если при `npm run sync:ios` появляется **`npm error could not determine executable to run`** — в этом каталоге нет `node_modules` (не запускали `npm install`, открыли не ту папку или удалили зависимости). Выполни **`npm install`** и повтори.

Если каталог `ios/` ещё не создан (редко для этого репозитория), сначала: `npx cap add ios`, затем снова `npm run sync:ios`.

3. Локально проверить веб без Xcode:

```bash
npm run dev
```

Открыть: `http://localhost:4173`

## Сборка на свой iPhone через Xcode (кабель)

В Xcode:

1. Открыть проект: `npm run open:ios` или `ios/App/App.xcodeproj`.
2. Target **App** → **Signing & Capabilities**: выбрать **Team** (Apple ID с программой разработчика или бесплатный аккаунт для личного устройства).
3. При необходимости сменить **Bundle Identifier** (тогда же обнови `appId` в `capacitor.config.ts` и приложение в App Store Connect).
4. Вверху выбрать свой **iPhone** и нажать **Run** (▶).

На **iPhone** в портрете — см. `Info.plist` и `README.md`. В браузере ландшафтная вёрстка описана в `web/styles.css`.

## TestFlight: внешние бета-тестеры (установка «как у людей»)

Нужен **Apple Developer Program** (платная учётная запись разработчика). Дальше — в общих чертах; детали интерфейса App Store Connect со временем меняются.

### 1. Приложение в App Store Connect

1. Зайти в [App Store Connect](https://appstoreconnect.apple.com/) → **Мои приложения** → **+** → новое приложение.
2. Платформа: iOS. **Bundle ID** должен **точно совпадать** с тем, что в Xcode (**Product** → выбери target → **Build Settings** → `PRODUCT_BUNDLE_IDENTIFIER`), сейчас это `com.tsvetpet.metorelic.game`.
3. Заполнить название, язык, SKU (внутренний идентификатор), пользовательское соглашение при необходимости.

### 2. Синхронизация веба и версия сборки

В каталоге проекта:

```bash
npm install
npm run sync:ios
```

Перед **каждой** загрузкой в TestFlight увеличь **сборку**:

- В Xcode: target **App** → **General** → **Version** (для пользователей, например `1.0.0`) и **Build** (целое число, монотонно: `1`, `2`, `3`…). Либо в **Build Settings**: `MARKETING_VERSION` и `CURRENT_PROJECT_VERSION`.

Дубликат «той же сборки» (тот же build number) App Store Connect не примет.

### 3. Архив и загрузка

1. В Xcode выбери **Any iOS Device (arm64)** или **Generic iOS Device** (не симулятор).
2. **Product** → **Archive**.
3. В **Organizer** выбери архив → **Distribute App** → **App Store Connect** → **Upload** → пройди шаги подписи (обычно **Automatically manage signing**).

Дождись обработки билда (письмо от Apple или статус в App Store Connect — часто 5–30 минут, иногда дольше).

### 4. TestFlight

1. В App Store Connect открой приложение → **TestFlight**.
2. **Внутреннее тестирование**: до 100 человек из твоей команды (роли в App Store Connect) — можно включить быстро, без отдельной модерации бета для первого внутреннего билда (правила уточняй в актуальной документации Apple).
3. **Внешнее тестирование**: создай группу, добавь **email** тестеров или публичную ссылку-приглашение (если доступно в твоём типе аккаунта). Для **первого** внешнего билда часто требуется короткая **Beta App Review** и ответы на вопросы (например, экспорт шифрования — для простой игры без своего шифрования обычно подходит стандартный ответ про только стандартное HTTPS Apple).

Тестеры устанавливают приложение **TestFlight** из App Store, принимают приглашение и ставят **Metro Relic** оттуда.

### 5. Частые проблемы

- **Signing**: неверный Team или профиль — проверь **Signing & Capabilities** и наличие приложения с тем же Bundle ID в App Store Connect.
- **Сборка не появляется**: подожди обработки; проверь почту на ошибки проверки.
- **Тестер не видит билд**: его email добавлен в группу, билд для этой группы **включён**, тестер принял приглашение в TestFlight.

## Альтернатива без TestFlight

- **Ad Hoc** или **Development**: ограниченный список UDID устройств, ручная выдача `.ipa` — неудобно для «случайных» внешних тестеров.
- Для широкой внешней беты на iPhone у Apple по сути рабочий путь — **TestFlight** (или публичный релиз в App Store).
